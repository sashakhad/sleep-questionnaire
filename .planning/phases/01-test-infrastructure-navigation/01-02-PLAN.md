Replace the current `navigateToSection` implementation:

Current (fragile):
```typescript
Cypress.Commands.add('navigateToSection', (section: string) => {
  cy.visit(`/dev?section=${section}`);
  cy.get('form', { timeout: 10000 }).should('exist');
  cy.wait(500);
});
```

New (robust):
```typescript
Cypress.Commands.add('navigateToSection', (section: string) => {
  cy.visit(`/dev?section=${section}`);
  cy.get('form', { timeout: 10000 }).should('exist');
  // Wait for the section title to render in the card header, confirming
  // React hydration and section routing are complete
  cy.get('[data-slot="card-title"]', { timeout: 10000 }).should('be.visible');
});
```

The key change: replace `cy.wait(500)` with an assertion that the card title element is visible. This confirms:
1. The page loaded (`cy.visit`)
2. The form hydrated (`cy.get('form')`)
3. The correct section rendered (card title visible)

Verify the actual card title selector by checking shadcn's `CardTitle` output â€” it uses `data-slot="card-title"` in recent shadcn versions. If the DOM doesn't use `data-slot`, fall back to checking for the section title text using something like `cy.contains('h1, h2, h3', SECTION_TITLES[section])` by importing from `test-data.ts`.

<task type="auto">
  <name>Task 2: Add form interaction helper commands</name>
  <files>cypress/support/commands.ts</files>
  <action>
Add new custom commands to `cypress/support/commands.ts` for common form interactions. Update the `Chainable` interface declaration to include them.

Commands to add:

1. `cy.selectOption(label: string, value: string)` â€” finds a shadcn Select trigger (`button[role="combobox"]`) associated with a label or containing text, clicks it to open the dropdown, then clicks the matching option. The pattern is:
   - Find the form item containing `label` text
   - Click the combobox button within it
   - Find `[role="option"]` containing `value` text and click it
   - Avoid `{ force: true }` unless needed for hidden elements

2. `cy.clickRadio(value: string)` â€” clicks a radio button by its `value` attribute: `cy.get(\`button[role="radio"][value="${value}"]\`).click({ force: true })`. Force is needed because shadcn RadioGroup items may be visually styled but technically hidden inputs.

3. `cy.checkCheckbox(label: string)` â€” finds a checkbox by its adjacent label text and checks it if not already checked. Pattern: find the label text, then find the sibling/parent checkbox element.

4. `cy.uncheckCheckbox(label: string)` â€” same as above but unchecks.

Keep all commands focused and simple. Each should be 5-15 lines max. Add proper TypeScript declarations in the `Chainable` interface for all new commands.

Avoid over-engineering â€” these are convenience wrappers around common patterns already used in `round3-feedback.cy.ts`. If a pattern from `round3-feedback.cy.ts` already works well (like `cy.contains('button', 'text').click()`), keep the command implementation consistent with that approach.
