<task type="auto">
  <name>Task 2: Create reusable assertion helpers</name>
  <files>cypress/support/assertions.ts, cypress/support/e2e.ts</files>
  <action>
Create `cypress/support/assertions.ts` with exported helper functions:

1. `assertSectionVisible(sectionTitle: string)` ‚Äî asserts the card header contains the given section title text. Uses `cy.contains()` or `cy.get()` on the card title element.

2. `assertProgressBar(expectedPercent: number)` ‚Äî asserts the progress bar's `aria-valuenow` attribute equals `expectedPercent` (shadcn `Progress` renders with `role="progressbar"` and `aria-valuenow`).

3. `assertStepIndicator(currentStep: number, totalSteps: number)` ‚Äî asserts the step text shows "Step {currentStep} of {totalSteps}".

4. `assertContinueEnabled()` / `assertContinueDisabled()` ‚Äî asserts the Continue button is enabled or disabled. Use `cy.contains('button', 'Continue')` to find the button, then check `.should('not.be.disabled')` / `.should('be.disabled')`.

5. `assertPreviousVisible()` / `assertPreviousNotVisible()` ‚Äî asserts the Previous button exists/is visible or is hidden (on the intro section it has `invisible` class).

6. `assertFormExists()` ‚Äî shorthand for `cy.get('form').should('exist')`.

All helpers should be pure Cypress chain functions (no return values needed ‚Äî they assert inline). Export as named functions.

Update `cypress/support/e2e.ts` to import the new modules:
```
import './commands';
import './test-data';
```
