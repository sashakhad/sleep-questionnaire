**Approach for empty-state testing:** Use the dev route (`cy.navigateToSection('demographics')`) which starts pre-filled with valid data and Continue enabled. Then clear individual fields to test that each required field gates the Continue button. This is more reliable than navigating from `/` through intro since it isolates demographics validation.

Tests to write:

1. **Continue is enabled with pre-filled valid data** — Navigate to demographics via dev route, verify Continue is enabled (baseline sanity check).

2. **Continue is disabled when yearOfBirth is cleared** — Navigate to demographics, clear the yearOfBirth combobox value (click the combobox trigger, then select a mechanism to clear it — or programmatically reset via the form). Since the YearComboboxField is a combobox, the most reliable approach is to navigate to the section without prefill. Instead, visit `/` (main route), check the disclaimer to enable Continue, click Continue to reach demographics, then verify Continue is disabled on demographics (all fields empty).

Revised approach — use main route for clean state:

1. **All fields empty — Continue is disabled** — Visit `/`, check disclaimer, click Continue to reach demographics. Verify Continue is disabled (no fields filled yet).

2. **Only yearOfBirth filled — Continue still disabled** — From demographics (empty), select a birth year via the combobox, verify Continue is still disabled (sex and zipcode missing).

3. **yearOfBirth and sex filled — Continue still disabled** — Also select sex, verify Continue is still disabled (zipcode missing).

4. **Short zipcode (< 5 chars) — Continue still disabled** — Type a 4-char zipcode, verify Continue is still disabled.

5. **All required fields filled — Continue is enabled** — Clear and type a 5+ char zipcode, verify Continue is enabled.

6. **Birth year combobox does not offer years making respondent younger than 12** — Open the birth year combobox, verify the most recent year available is `currentYear - 12`. The combobox should NOT contain `currentYear - 11` or later.

7. **Pre-filled demographics on dev route — Continue is enabled** — Navigate via dev route, verify Continue is enabled and all fields are populated.

For navigating from the main route through intro to demographics: visit `/`, find and click the disclaimer checkbox, then click the Continue button. The form should advance to demographics.

Use `cy.contains('button', 'Continue')` for clicking Continue, and standard Cypress retry for assertions.
