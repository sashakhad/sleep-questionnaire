**Endpoint contract:**
- POST /api/generate-pdf
- Body: `{ data: QuestionnaireFormData, userName?: string }`
- Success: returns `application/pdf` with Content-Disposition attachment header
- Failure: returns JSON `{ error: string }` with status 500
- NOT auth-protected (middleware matcher only covers /admin/:path* and /api/responses/:path*)
- Server validates `body.data` with `questionnaireSchema.parse()`

**Mock data source:**
The MOCK_DATA object in `src/components/questionnaire/QuestionnaireForm.tsx` (lines 16-159) contains valid questionnaire data that passes Zod validation. Copy its structure for the test payload — do NOT import it (Cypress runs outside the app bundle).

**Key mock data fields needed for a valid payload:**
- intro: { acceptedDisclaimer: true }
- demographics: { yearOfBirth: 1990, sex: 'male', zipcode: '10001', weight: 165, height: 70 }
- All other sections with valid values (scheduledSleep, unscheduledSleep, breathingDisorders, restlessLegs, parasomnia, nightmares, chronotype, sleepHygiene, bedroom, lifestyle, mentalHealth, sleepDisorderDiagnoses, daytime)

**Established patterns:**
- Use `cy.request()` for API testing
- Use `failOnStatusCode: false` when testing error responses
