  <action>
Create `cypress/e2e/api-csv.cy.ts` with the following test structure:

```
describe('CSV Export Endpoint (GET /api/responses/csv)', () => {
  describe('Authentication', () => {
    - returns 401 when no admin session cookie is present
    - returns CSV when authenticated with valid admin session
  })

  describe('Response Format', () => {
    beforeEach: cy.loginAsAdmin()

    - returns Content-Type text/csv
    - returns Content-Disposition header with filename containing date
    - response body is non-empty
    - first line contains expected CSV column headers (id, year_of_birth, sex, zipcode, weight_lbs, height_inches, submitted_date, submitted_time)
    - CSV rows are comma-separated
  })
})
```

Key implementation details:
- Use `cy.request({ url: '/api/responses/csv', failOnStatusCode: false })` for the 401 test
- Use `cy.loginAsAdmin()` before authenticated tests (in a `beforeEach`)
- Parse `response.headers['content-type']` to check for 'text/csv'
- Parse `response.headers['content-disposition']` to check for 'attachment' and '.csv'
- Split `response.body` by newline, check first line contains expected headers
- The database may or may not have responses — tests should work with zero or more rows (just verify headers exist on the first line)
- Do NOT test row data content (depends on DB state) — only test structure and format
